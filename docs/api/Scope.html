Scope Class

Defined in: lib/Scope.js

A scope representing a React component instance and a bunch of DOM nodes.

NOTE

You do not have to use this interface directly. Instead, use the primary drill export.

Constructor Documentation

Scope(component: React.Component, nodes: Array.<Node>, path: String, parentScope: Scope)

Parameters (4)

component
React.Component

A rendered instance of a React.Class type.

nodes
Array.<Node>

The DOM nodes this scope represents. Defaults to the component's root DOM node.

Defaults to: null

path
String

The path of this scope. Defaults to the displayName of the component's type.

Defaults to: null

parentScope
Scope

The parent scope in which this scope resides. Null if this is the global scope.

Defaults to: null

Example

var Scope = require('react-drill/lib/Scope');
var myComponent = ReactDOM.render(...);
var scope = new Scope(myComponent);

  .click()

    Example

    function click(node) {
  // do something to trigger a click on this node
  console.log(node.tagName + ' was clicked.');
}

Scope.registerHTMLElementMethod('click', click);

Scope(MyComponent).find('button').click();
// => "BUTTON was clicked."

Static Property Documentation

  • component: React.Component

    A cursor to the React component instance this scope represents. Component selectors will be operating based on this instance.

  • node: Node

    The first node of this scope. Most of the time, we only have one node to operate on (like a React component's root node) and so this is a convenient accessor.

  • nodes: Array.<Node>

    The DOM nodes this scope contains. All DOM helpers will be operating against these nodes.

  • parentScope: Scope

    The parent scope for this scope, if any. We utilize this for things like #end() where we need to traverse the scope ancestry.

  • path: String

    We use the path for displaying helpful assertion errors.

    The path could look like:

    Root > Button > .icon
Root > .icon[] > .icon-close

    find(selector: String|React.Class, cond: Function) -> Scope

    Find a single component or DOM node within this scope.

    This is a convenience accessor which delegates to either Scope#findByType or Scope#findBySelector based on the selector you pass.

    WARNING

    In order for React types to be properly identified, we assume they have a displayName attribute, which is most likely present if you're using any JSX pre-processor like babeljs or such.

    Parameters (2)

    selector
    		String|React.Class

    Either a querySelector or a React.Class;

    cond
    		Function

    Predicate to locate the exact target.

    Return Value

    Scope

    A new scope representing either the new component that was located, or the new DOM node.

    has(selector: Selector|React.Class, cond: Function) -> Boolean
